初识 Docker

Docker 概念

  • Dokcer 是一个开源的应用容器引擎
  • 基于 Go 语言实现
  • 容器完全使用沙箱机制，相互隔离
  • 容器性能开销极低

Docker 安装

  • 脚本安装

Docker 架构

  • 镜像（image）：相当于是一个 root 文件系统
  • 容器（container）：镜像和容器的关系，就像是面向对象程序设计中的类和对象一样，镜像是静态的定义，容器是镜像运行时的实体。容器可以被创建、启动、停止、删除、暂停等
  • 仓库（repository）：仓库看作是一个代码控制中心，用来保存镜像

Docker 数据卷

数据卷概念

  • 数据卷是宿主机中的一个目录或文件
  • 当容器目录和数据卷绑定之后，两者的修改是同步的
  • 一个数据卷可以被多个容器挂载，一个容器也可以挂载多个数据卷
  • 作用
    • 容器数据持久化
    • 外部机器和容器间接通信
    • 容器之间数据交换

数据卷配置

  • 创建容器时，设置数据卷

      2. 如果目录不存在，则自动创建
      3. 可以挂载多个数据卷
